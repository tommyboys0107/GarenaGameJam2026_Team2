# Project Structure

```
GarenaGameJam2026_Team2/
├── Assets/                    # All game content lives here
│   ├── Scenes/                # Unity scenes (.unity files)
│   │   └── SampleScene.unity  # Main/default scene
│   ├── Settings/              # URP and rendering configuration
│   │   ├── UniversalRP.asset  # URP pipeline asset
│   │   ├── Renderer2D.asset   # 2D Renderer settings
│   │   └── Scenes/            # Scene templates
│   ├── InputSystem_Actions.inputactions  # Input action maps (Player: Move, Look, Attack)
│   └── DefaultVolumeProfile.asset        # Post-processing volume
├── Packages/                  # Unity Package Manager config
│   └── manifest.json          # Package dependencies
├── ProjectSettings/           # Unity project settings (do not manually edit)
├── Library/                   # Unity cache (gitignored, auto-generated)
├── .gitignore                 # Comprehensive Unity gitignore
└── .kiro/                     # Kiro AI assistant config
    ├── settings/              # MCP and tool configuration
    └── steering/              # Steering documents (this file)
```

## Conventions

- **Scripts** should go under `Assets/Scripts/` (create as needed, organized by feature)
- **Art assets** should go under `Assets/Art/` or `Assets/Sprites/`
- **Prefabs** should go under `Assets/Prefabs/`
- **Audio** should go under `Assets/Audio/`
- **ScriptableObjects** should go under `Assets/Data/` or alongside their feature
- Every new folder or asset must have a corresponding `.meta` file (Unity generates these automatically)
- Do not manually edit files in `ProjectSettings/` or `Library/`
- The `Library/` folder is gitignored and regenerated by Unity

## Scene Organization

- Primary scene: `Assets/Scenes/SampleScene.unity`
- New scenes should be added to `Assets/Scenes/`
- Register scenes in Build Settings when needed for builds
